Exercise 1: GitHub Setup and Introduction Page

We'll be using the GitHub developer platform to host your personal class website - to be used to store and present all exercises and projects. GitHub websites are stored in a "repositories" which you can think of as a folder on your computer.

  1. Create an account on GitHub (make sure to verify the account) using your Fordham email.
  2. After you're logged in, go to your main repositories page. Click the NEW button.
  3. Name your repository whatever you would like (something like “class-site” if you can’t think of anything).
  4. Select PUBLIC (Anyone on the internet can see this repository).
  5. Check Add a README file.
  6. Leave other options as-is.
  7. Click CREATE REPOSITORY.
  8. You are now viewing the contents of the repository which are currently empty except for the README file. A README file is simply a description of the contents of the respository. Click the edit icon if you'd like to change the text of the README file.

In order for your repository to be treated and rendered like a website, you need to initialize GitHub Pages on this repository. Github pages is a feature that makes it easy and free to host handmade websites.

  1. Click SETTINGS in the top navigation.
  2. Click PAGES in the left navigation.
  3. Under BRANCH, select MAIN. Click the SAVE button.
  4. Wait a few minutes and refresh the page. You will see your repo link: username.github.io/class-site

As you complete exercises and projects, you will add these files to the repo. In this exercise, we will create an introduction homepage (index.html) but going forward, the process is:

  1. Click the ADD FILE > UPLOAD FILES button at the top of the repository.
  2. Drag the new files or folders to the area indicated.
  3. COMMIT CHANGES - this is where you add some short description of what you've changed to keep track of your activity (something like "first upload", "updated homepage", "add new project", etc.)
  4. Wait a few minutes and then check your repo link.
  5. Add your URL to the spreadsheet.

Note: GitHub has many features, including a desktop app that automatically updates your repo. We will not be using the app but feel free to try out these features on your own computer.

Exercise 2: Text and Images

Use this text to recreate Ten Principles for Good Design by Dieter Rams. How can you communicate using text and images? Use what you know about HTML and CSS (also, refer to the resources) and follow these instructions:

  • One page only, a stylesheet and a folder for images
  • Include one image for each of the 10 principles which demonstrates the principle in your opinion (be creative, do not use Dieter Ram images, you can also create your own images). These can be placed anywhere that makes sense within your design. Resize your images using the width or height attributes in HTML or use CSS; make sure they are not distorted.
  • Use as many HTML tags and CSS properties as you'd like.
  • In terms of design, think about what the choice of fonts, colors and placement of images says about what you're communicating.
  • Remember this is an exercise - do your best, even if its not perfect!

Recommended Process:

  1. Create a local folder "exercise2". Folder should contain index.html, a stylesheet and a sub-folder for images.
  2. Upload this folder to your GitHub class website. The URL should look something like: username.github.io/class-site/exercise2/. Add the URL to the spreadsheet.

Exercise 3: Interactive

Pick a song lyric of any length and design a one-page website that uses hover effects, transitions, transform, and/or animation. You can choose to do this purely typographically or add other graphics as necessary. Think about what elements are interactive vs. animated. What visual cues can you include in the design to let the user know to hover or click on something? Be expressive and experiment!

Examples: Flower, Felt, Bee, Sunburn

Exercise 5: Responsive

Create an experimental, responsive, one-page website that changes across different screen sizes using media queries.

  • from day to night
  • breakfast then lunch then dinner
  • rain then sun then snow
  • small to medium to large

do colors change? does type change size? do things appear or disappear? is there animation? how might your design communicate different feelings or meanings when it adapts?

Examples: Jacob Hoving, Cow That, In a Week, Caterpillar, Tea Party
