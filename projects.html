Project 1: One-Page Website - Due Oct 3

Make a one-page website about your favorite book. Use this opportunity to use your beginner knowledge of HTML and CSS. You can include text, images, animated gifs, reviews, characters, your personal thoughts, original images you create, etc. What is the purpose of your website (to sell the book, provide spoilers, summarize the plot, visualize the book using my own art, review the book)? Be flexible with the presentation and design - you will likely NOT be able to do what you envision but work within those limitations. As in all projects, experiment!

Requirements:

  • One page only (index.html), a stylesheet and a folder for images and other assets.
  • Use as many HTML tags and CSS properties as you'd like.
  • Websites do not have to be responsive, you can adjust the browser to the size you need for the presentation.

Recommended Process:

  1. Think about what kind of content you'd like to include and start gathering it. If using text from another website, please source and provide a link.
  2. Create a local folder "project1". Folder should contain index.html, a stylesheet and a sub-folder for images and other assets.
  3. When the project is complete, upload this entire folder to your GitHub class website. The link should look something like: github-username.github.io/class-site/project1/. You can continue to edit your project and re-upload.
  4. Add your URL to the spreadsheet.

Project 2: Website Critique - Due Oct 17

Choose a website or app you use frequently or have used recently. Exclude social media such as IG, Facebook, TikTok, X, WhatsApp, etc. and default apps on your devices such as Photos, Mail, Camera, Weather, etc.

Critique the chosen website or app in terms of content, visual appeal/UI, and interactivity/UX - use this slideshow for guidelines. More guidelines are below but you do not need to answer every question Your critique should be presented as a website, +/-1000 words, and include screenshots or other graphics/video to illustrate your critique. Keep the design simple - this project is about how well you communicate and express ideas, how observant you are, how you can translate a website experience. There’s no better way to articulate design than with writing.

  • Introduction: What type of website is this (corporate, e-commerce, journalism, artist, blog, etc.) and what is its purpose? Examples: Does it aim to inform people about the company or organization, to advertise a product, to provide a means for online purchase, to recruit new clients/volunteers, to convince people of a particular cause? Who is the target audience?
  • Content: Does the content contribute to the site’s purpose? Can you easily read the text? Are the images/media engaging?
  • Visual Appeal/UI: Is there a visual hierarchy and how does the design accomplish it? Is there consistency with colors, fonts, layouts? Are there other engaging visual elements? Are the typefaces appropriate? Does it work well on other devices?
  • Interactivity/UX: Does the homepage direct you where to go or what to do? Does the site provide easy navigation to help you find content? What are other ways you interact with this website or app? Is it an enjoyable experience overall?
  • Evaluation / Recommendations: What could be improved and how?

Recommended Process:

  1. Create a local folder "project2". Folder should contain index.html, a stylesheet and a sub-folder for images and other assets.
  2. When the project is complete, upload this entire folder to your GitHub class website. The link should look something like: github-username.github.io/class-site/project2/. You can continue to edit your project and re-upload.
  3. Add your URL to the spreadsheet.
  4. 1-2 volunteers will present in class.

Project 3: Narrative - Due Nov 21

Create a website that tells a story. You could use an existing short story, fairy tale, poem. Or, you can design a personal narrative: 24 hours in my day, my favorite artists, how to make my mom's chocolate cake, why I love Friends tv show, etc.

Requirements:

  • Multiple pages or a single page.
  • Use text, images, or videos.
  • Think about the various ways users can access parts of the story - do they need to follow a set path (linear) or can they view the story in a random order (non-linear)? How will you lead them there? Also consider how your design will help the user figure out what to click on.
  • Your website could be informative, artistic, or experimental. What’s important is that it demonstrates how the web can be used to communicate in a linear or non-linear way.
  • Bonus if your website is responsive!

Recommended Process:

  1. Choose an existing story or write/compose the story. It could be written from beginning to end or you could write it as a series of ideas with no beginning or end.
  2. Create a site map to figure out how the pages are connected.
  3. Sketch out page layout ideas. Gather or create assets such as images or videos.
  4. Refer to the slideshow for inspo.
  5. Design with html/css in a local folder "project3". Folder should contain html pages, a stylesheet and a sub-folder for images and other assets.
  6. When the project is complete, upload this entire folder to your GitHub class website. The link should look something like: username.github.io/class-site/project3/. You can continue to edit your project and re-upload.
  7. Add your URL to the spreadsheet.
  8. Project 3 will be presented in class. If your project is late, you will not be able to present it which will affect your grade.
