Project 1: One-Page Website - Due Feb 24

Make a one-page website about your favorite book. Use this opportunity to use your beginner knowledge of HTML and CSS. You can include text, images, animated gifs, reviews, characters, your personal thoughts, original images you create, etc. What is the purpose of your website (to sell the book, provide spoilers, summarize the plot, visualize the book using my own art, review the book)? Be flexible with the presentation and design - you will likely NOT be able to do what you envision but work within those limitations. As in all projects, experiment!

Requirements:

  • One page only (index.html), a stylesheet and a folder for images and other assets.
  • Use as many HTML tags and CSS properties as you'd like.
  • Websites do not have to be responsive, you can adjust the browser to the size you need for the presentation.

Recommended Process:

  1. Think about what kind of content you'd like to include and start gathering it. If using text from another website, please source and provide a link.
  2. Create a local folder "project1". Folder should contain index.html, a stylesheet and a sub-folder for images and other assets.
  3. When the project is complete, upload this entire folder to your GitHub class website. The link should look something like: github-username.github.io/class-site/project1/. You can continue to edit your project and re-upload.
  4. Add your URL to the spreadsheet.

Project 2: Narrative - Due Apr 7

Create a website that tells a story. You could use an existing short story, fairy tale, poem. Or, you can design a personal narrative: 24 hours in my day, my favorite artists, how to make my mom's chocolate cake, why I love Friends tv show, etc. Consider the UI and UX aspects of the website.

Requirements:

  • Multiple pages or a single page.
  • Use text, images, or videos.
  • Think about the various ways users can access parts of the story - do they need to follow a set path (linear) or can they view the story in a random order (non-linear)? How will you lead them there? Also consider how your design will help the user figure out what to click on.
  • Your website could be informative, artistic, or experimental. What’s important is that it demonstrates how the web can be used to communicate in a linear or non-linear way.
  • Bonus if your website is responsive!

Recommended Process:

  1. Choose an existing story or write/compose the story. It could be written from beginning to end or you could write it as a series of ideas with no beginning or end.
  2. Create a site map to figure out how the pages are connected.
  3. Sketch out page layout ideas. Gather or create assets such as images or videos.
  4. Design with html/css in a local folder "project2". Folder should contain html pages, a stylesheet and a sub-folder for images and other assets.
  5. When the project is complete, upload this entire folder to your GitHub class website. The link should look something like: username.github.io/class-site/project2/. You can continue to edit your project and re-upload.
  6. Add your URL to the spreadsheet.
  7. Project 2 will be presented in class. If your project is late, you will not be able to present it which will affect your grade.
